프론트엔드 FrontEnd

웹에서 사용자에게 시각적으로 보여지는 부분을 의미한다.
프론트엔드쪽에서는 주로 HTML, CSS, 자바스크립트가 쓰이며, 요즘은 페이스북에서 개발한 자바스크립트에서 파생된 싱글 페이지 애플리케이션 형태의 리액트가 많이 사용된다.

HTML

웹사이트의 모습을 기술하기 위한 마크업 언어

CSS

HTML 등의 마크업 언어로 작성된 문서가 실제로 웹사이트에 표현되는 방법을 정해주는 스타일 시트 언어

JavaScript

Ecma International의 프로토타입 기반의 프로그래밍 언어

React

메타에서 개발한 오픈 소스 자바스크립트 라이브러리

자체적인 만들어보는 독학 페이지

  1. Node.js

    2. 단일 처리 시스템으로 알고 있지만 대신 그 만큼 서버 확장은 용이하다고 한다. JavaScript를 사용한다. 프레임워크 마냥 소개하는 경우도 많은데 정확히 그건 아니고 정확히는 JavaScript런타임이라한다.

    런타임이란?
    운영체제 위 또는 운영체제 자체에서 실행되면서 특정 프로그래밍 언어가 구동될 수 있는 환경

  2. Spring boot

    3. Java 기반 프레임워크, 그냥 Spring과 다르게 개발자가 설정 파일을 작성하거나 컨테이너를 구성하거나 할 필요가 없다.(둘이 착각X)
    Node.js와 달리 단일 처리(즉,Single Thread)가 아닌 다중 처리(즉,Multi Thread)인 것으로 알고 있다.
    하지만 배우는 난이도가 좀더 어렵다고 한다.

  3. Nest.js

    4. Node.js 런타임 위에서 동작하는 TypeScript용 오픈 소스 백엔드 웹 프레임워크.(나무위키 참고)
    설명으로는 Node.js를 사용하는데 정해진 아키텍처가 없으니 개발자마다 다른 구조로 작성하게 되는데 이로 인해서 협업이나 유지 보수가 어려우니
    이걸 보완하기 위해서 나온 것이라고 한다.
